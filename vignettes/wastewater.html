Introduction

Wastewater-based epidemiology (WBE) monitors the concentration of pathogen genetic material — such as SARS-CoV-2 RNA — in municipal wastewater. Because viral shedding into sewage can precede clinical case reporting by days to weeks, surveillance networks that continuously sample and sequence wastewater at multiple sites can provide an early-warning signal of emerging or resurging outbreaks, and identify which sites are effected in order to triage resources.

From a statistical perspective, a national WBE network yields a multivariate data stream: at each time point \(t\) (here, each epidemiological week), we observe a measurement \(X_{tk}\) at each of \(K\) monitoring sites. The surveillance task has two components:

  1. Change-point detection — raise an alarm as soon as the data depart from a baseline regime, controlling a type-I error criterion such as the probability of false alarm (PFA) or the average run length under the null (ARL).

  2. Stream identification — after (or concurrent with) an alarm, determine which sites \(k\) are driving the change, so that public-health resources can be directed appropriately.

The multichangepoints package is designed for exactly this setting. A univariate test supermartingale (TSM) is built for each site, marginal increment sequences are combined across sites (via ProductCombiner, AverageCombiner, or UniversalPortfolioCombiner), and a single ShiryaevRobertsDetector monitors the combined evidence sequentially — stopping as soon as the evidence crosses a threshold that guarantees the desired error control.

Data

The dataset wastewater_aggregate.csv is published by the Public Health Agency of Canada, and aggregating weekly wastewater measurements from monitoring sites across Canada.
Each row contains one observation indexed by site \(\times\) week \(\times\) pathogen. Key columns:

Column Description
site Monitoring location
province Canadian province / territory
weekstart Monday of the epidemiological week
measureid Pathogen marker (covN2, fluA, fluB, rsv)
w_avg Weekly average viral signal (copies / mL, log10-transformed at source)

The wave structure is broadly synchronous across provinces — consistent with a national-level change in pathogen prevalence — but the magnitude and timing of individual site peaks vary, reflecting local population sizes, sampling frequency, and catchment characteristics. This heterogeneity motivates a multi-stream approach: rather than pooling sites into a single signal, the multichangepoints framework tests each site sequentially and combines the evidence in a principled way, allowing the detector to remain powerful even when only a subset of sites shows an elevated signal.

Missing Data

Some sites are missing weeks, wherein COVID-19 measurements were not taken.

missing_by_site <- cov %>%
  group_by(site) %>%
  summarize(
    first_week = min(weekstart),
    last_week  = max(weekstart),
    observed   = list(sort(unique(weekstart))),
    .groups    = "drop"
  ) %>%
  rowwise() %>%
  mutate(
    expected           = list(seq(first_week, last_week, by = "7 days")),
    n_expected_observed = length(expected),
    n_missing          = sum(!expected %in% observed),
    pct_missing        = round(100 * n_missing / n_expected_observed, 1)
  ) %>%
  ungroup() %>%
  select(site, n_expected_observed, n_missing, pct_missing) %>%
  arrange(desc(pct_missing))

Change-point detection for WBE

# checking data transformations that lead to independent and bounded innovations
cov_alberton <- cov %>% 
  select(site, weekstart, w_avg) %>%
  filter(site == "Alberton") %>% 
  arrange(weekstart) %>%
  mutate(lag_w_avg = lag(w_avg)) %>%
  mutate(log_change = log1p(w_avg) - log1p(lag_w_avg))
