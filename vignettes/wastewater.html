Introduction

Wastewater-based epidemiology (WBE) monitors the concentration of pathogen genetic material — such as SARS-CoV-2 RNA — in municipal wastewater. Because viral shedding into sewage can precede clinical case reporting by days to weeks, surveillance networks that continuously sample and sequence wastewater at multiple sites can provide an early-warning signal of emerging or resurging outbreaks, and identify which sites are effected in order to triage resources.

From a statistical perspective, a national WBE network yields a multivariate data stream: at each time point \(t\) (here, each epidemiological week), we observe a measurement \(X_{tk}\) at each of \(K\) monitoring sites. The surveillance task has two components:

  1. Change-point detection — raise an alarm as soon as the data depart from a baseline regime, controlling a type-I error criterion such as the probability of false alarm (PFA) or the average run length under the null (ARL).

  2. Stream identification — after (or concurrent with) an alarm, determine which sites \(k\) are driving the change, so that public-health resources can be directed appropriately.

The multichangepoints package is designed for exactly this setting. A univariate test supermartingale (TSM) is built for each site, marginal increment sequences are combined across sites (via ProductCombiner, AverageCombiner, or UniversalPortfolioCombiner), and a single ShiryaevRobertsDetector monitors the combined evidence sequentially — stopping as soon as the evidence crosses a threshold that guarantees the desired error control.

The wave structure is broadly synchronous across provinces — consistent with a national-level change in pathogen prevalence — but the magnitude and timing of individual site peaks vary, reflecting local population sizes, sampling frequency, and catchment characteristics. This heterogeneity motivates a multi-stream approach: rather than pooling sites into a single signal, the multichangepoints framework tests each site sequentially and combines the evidence in a principled way, allowing the detector to remain powerful even when only a subset of sites shows an elevated signal.

Change-point detection for WBE

Change-point detection with WBE data is a major challenge because it is very difficult to identify the state of control that is supposed to be represented by the pre-change model. In this case, the state of control is a site-specific simple or composite model from which the observed sequence is generated when there is no epidemic. Since we have multiple sites, we might try to make some kind of ‘synthetic state of control’ by synthesizing data across sites that were epidemic-free at various times. Unfortunately, this is subject to the many uncertainties inherent in wastewater surveillance across multiple sites Joung et al (2023), including:

  • Target quantity: The true target quantity of interest is the average viral load in a particular population week over week. This quantity varies naturally over time when the virus is in an endemic state. While this variation is not genuinely random, it can be modeled as such. For instance, we might assume it maintains some low-level average with as-if-Gaussian fluctuations from week to week, in which case the sequence is stationary in the endemic phase.
  • Sampling: The methods of sample collection, shipping, and storage (compositing vs grab sampling, location, timing, shipping temperature, distance to lab, missing data), which may vary across sites and time.
  • Analysis: Laboratory analysis variability (de-gritting subsampling, homogenization, instrument error, human error).
  • Infrastructure: The location and engineering of sewage systems and wastewater treatment plants (catchment area size and population, combined or separate (‘sanitary’) collection of runoff and wastewater).
  • Weather: Weather conditions (rainwater or snowmelt contributing to influent, temperature, pH), which vary across sites and seasons, with some degree of stochasticity.

In the literature, these uncertainties are dealt with under the rubric of normalization, which is an active and un-clarified area of research. In some sense, defining a proper state of control from which deviations are meaningful is the whole challenge of WBE. It may be easier to define the state of control for certain sites, like those with sanitary sewage systems that are less influenced by major weather events.

For any given site, we could imagine an ideal way to establish its state of control via potential outcomes. In one potential outcome, the site experiences a COVID epidemic and in another epidemic the site experiences no COVID epidemic (or some kind of endemic state, both of these are very rough definitions). This is, of course, impossible in reality; we cannot even imagine a reasonable experiment because there is no way to manipulate the hypothetical intervention ‘COVID epidemic’. This could be considered a problem in observational causal inference. One way to establish the state of control is through some kind of matching, wherein we find pairs site that are similar along various axes likely to effect background rates (collection, engineering, weather, etc); but in one site there is no epidemic and in another site there is an epidemic. We could also do this by site-periods, with sites serving as their own control over periods where they are epidemic free. Another way is to fit models for the state of control. This is attempted below. In either case, establishing a meaningful state of control (a simple or composite pre-change model) should be done separately from running a change-point algorithm, which can detect deviations but cannot define the state of control without a user.

# 1. start with a single site with no missing data (Vancouver)
site_series <- cov %>%
  filter(site == "Vancouver Lions Gate") %>%
  arrange(weekstart) %>%
  mutate(y = log1p(w_avg)) %>%
  mutate(log_change = y - lag(y)) %>%
  filter(!is.na(log_change))

x <- site_series$log_change # time series is the log change in concentrations

# 2. Fit AR(2) using arima() (degree determined by AIC)
fit <- arima(x, order = c(2, 0, 0))
resid <- residuals(fit)

# Fit diagnostics
Box.test(resid, lag = 10, type = "Ljung-Box")
#> 
#>  Box-Ljung test
#> 
#> data:  resid
#> X-squared = 12.702, df = 10, p-value = 0.2408
par(mfrow = c(2, 2))
plot(resid, type = "l", main = "Residuals over time")
acf(resid, main = "ACF of residuals")
pacf(resid, main = "PACF of residuals")
qqnorm(resid); qqline(resid)

# point estimate of nuisance parameter 
sigma_hat <- fit$sigma2
model <- GaussianModel(
  mean_pre = 0,
  sd_pre = sigma_hat,
  mean_post = c(0, 1), # this captures a one-sided alternative, that the rate of change increases
  sd_post = sigma_hat,
  method = "predictable"
)
inc <- compute_increments(TSM(model), x, log = TRUE)
sr <- ShiryaevRobertsDetector(
  alpha = 1/52, # average run length without false alarm is 1 year
  criterion = "ARL")
det_out <- run_detector(sr, inc, log = TRUE)
par(mfrow = c(2, 2))

Joint wastewater site monitoring

To pass the wastewater data through the CPD algorithm, we need the site data as a matrix with NA where there are missing values.

# Full weekly spine: every 7 days from global min to global max
all_weeks <- seq(min(cov$weekstart), max(cov$weekstart), by = "7 days")

joint_matrix <- cov %>%
  mutate(log_change = log1p(w_avg) - log1p(lag(w_avg))) %>%
  select(site, weekstart, log_change) %>%
  complete(site, weekstart = all_weeks) %>%   # pad all site × week combinations with NA
  arrange(site, weekstart) %>%
  pivot_wider(names_from = weekstart, values_from = log_change) %>%
  column_to_rownames("site") %>%
  as.matrix()

cat("Dimensions: ", nrow(joint_matrix), "sites x", ncol(joint_matrix), "weeks\n")
