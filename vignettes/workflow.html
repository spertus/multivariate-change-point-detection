multichangepoints workflow

This vignette demonstrates the core workflow for sequential change-point detection with:

  • IID Gaussian data generating process (DGP)
  • simple-vs-simple test supermartingale (TSM)
  • Shiryaev-Roberts (S-R) detector
devtools::load_all("~/Dropbox/CANSSI/multi-change-points/")
#> ℹ Loading multichangepoints
#> Warning: package 'testthat' was built under R version 4.5.2
library(multichangepoints)
set.seed(20260305)

1. Specify a Gaussian model and TSM

# define a simple change model, including all pre-change and post-change parameters
model <- GaussianModel(
  mean_pre = 0,
  sd_pre = 1,
  mean_post = 1,
  sd_post = 1
)

# construct a tsm object from the model, it will receive data
tsm <- SimpleVsSimpleTSM(model)

2. Create an IID Gaussian DGP

# set up the data generating process for the data sequence, including the pre/post distributions and the change-point
dgp <- DGP(
  generator = default_gaussian_dgp,
  pre_params = list(mean = 0, sd = 1), # parameters pre-change
  post_params = list(mean = 1, sd = 1), # parameters post-change
  nu = 120, # change-point
  name = "iid-gaussian-change"
)

# generate data from the dgp
N <- 250
x <- generate_stream(dgp, N = N, K = 1)
head(x)
#> [1] -0.2698252 -0.9911324 -0.4595598 -0.8808831 -1.4012064  1.9237771

3. Build increments and run the S-R detector with ARL control

# increment sequence from data x
inc <- compute_increments(tsm, x)

# instantiate a detector class with ARL control
sr_1 <- ShiryaevRobertsDetector(
  alpha = 0.05, # average run length without false alarm is only 20
  criterion = "ARL"
)
sr_2 <- ShiryaevRobertsDetector(
  alpha = 0.001, # average run length without false alarm is 1000
  criterion = "ARL"
)


# push increments through detector sr
det_out_1 <- run_detector(sr_1, inc)
det_out_2 <- run_detector(sr_2, inc)
str(det_out_1)
#> List of 4
#>  $ statistic    : num [1:250] 0.463 0.329 0.509 0.379 0.206 ...
#>  $ stopping_time: int 14
#>  $ alarm        : logi TRUE
#>  $ criterion    : chr "ARL"

The detector returns:

  • statistic: the S-R path over time
  • stopping_time: first alarm time (or Inf)
  • alarm: whether an alarm occurred by the horizon

The following figure plots the path of the TSM and ALR-controlled S-R statistic, along with the alarm time.

4. Build increments and run the S-R detector with PFA control

# define a spending schedule
spend_1 <- dgeom(1:N, p = 0.1) # geometric spending schedule with p = 0.1
spend_2 <- dpois(1:N, lambda = 120) # poisson spending schedule centered at true nu
spend_1 <- spend_1/sum(spend_1) # normalize
spend_2 <- spend_2/sum(spend_2)  # normalize


# instantiate a detectors with PFA control
sr_pfa_1 <- ShiryaevRobertsDetector(
  alpha = 0.05,
  criterion = "PFA",
  spending = spend_1
)
sr_pfa_2 <- ShiryaevRobertsDetector(
  alpha = 0.05,
  criterion = "PFA",
  spending = spend_2
)


# push increments through detector sr
det_out_1 <- run_detector(sr_pfa_1, inc)
det_out_2 <- run_detector(sr_pfa_2, inc)
old_par <- par(mfrow = c(1, 2))

5. Estimate operating characteristics by simulation

sr <- ShiryaevRobertsDetector(
  alpha = 0.001,
  criterion = "ARL"
)

mc <- run_simulation(
  detector = sr,
  tsm = tsm,
  dgp = dgp,
  n_rep = 100,
  N = 300,
  seed = 77
)

mc
#>           detector                 dgp n_rep horizon  nu false_alarm_prob
#> 1 shiryaev-roberts iid-gaussian-change   100     300 120              0.1
#>      ARL   ADD
#> 1 125.12 10.12

This returns Monte Carlo summaries including:

  • false_alarm_prob: proportion of runs alarming before (or at) the change
  • ARL: average run length to alarm (censored at horizon)
  • ADD: average detection delay after the change
